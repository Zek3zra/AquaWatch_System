Project Objective

Tracking the availability of drinking water across a sprawling university campus presents a constant logistical challenge. Traditional manual checking is highly inefficient, often leaving students without access to water in critical areas.

The AquaWatch system was engineered specifically to solve this visibility issue. By utilizing localized weight measurements to determine volume, this architecture serves as a strict real-time visualization and monitoring tool. It provides students with an immediate, web-accessible map of optimal dispensers while giving administrators a centralized Command Center for rapid maintenance deployment.

Technical Architecture

Deep dive into the edge logic and cloud infrastructure governing the telemetry pipeline.

Strict Telemetry Monitoring

The system is designed purely as an active visualization tool. It completely isolates logic from complex inventory management systems, utilizing the ESP32 to focus purely on high-frequency, reliable weight conversion and secure HTTPS POST requests.

Over-The-Air Calibration

Administrators can push absolute zero-offset commands directly to the physical ESP32 flash memory via the web queue, saving the new offset permanently into the NVRAM without physical intervention.

WebSocket Sync

The front-end utilizes Supabase Realtime channels to inject database changes directly into the DOM, completely eliminating the need for heavy HTTP polling requests.
